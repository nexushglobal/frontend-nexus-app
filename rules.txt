MODULO DE ECOMMERCE

EL MODULO ECOMMERCE NOS PERMITIRA ADMINISTRAR TANTO LAS ORDENES COMO LOS PRODUCTOS DE LA PLATAFORMA
se divide en 2 grandes secciones la vista para admin con el rol de FAC y la vista para usuarios con el rol de CLI

nos centraremos en la vista para admin que a su vez tiene los siguientes modulos
-Listado de productos con filtros y paginacion (ya lo hicimos) -> /dashboard/(facturacion)/fac-ecommerce/productos
-Crear product (ya lo hicimos) -> /dashboard/(facturacion)/fac-ecommerce/productos/nuevo
-Detalle de producto (Pendiente) -> /dashboard/(facturacion)/fac-ecommerce/productos/detalle/:id

DETALLE DE PRODUCTO
1. lo primero que tenemos que hacer en consumir la api del action getProductAdminAction y getCategoriesAction
2. consultar el historial de stock del producto con el service getHistoryProductStock y el producto getProductAdmin
3. la informacion la presentaremos en un formulario editable, donde el usuario podra editar los siguientes campos (solo esos campos) con el action updateProductAction que es un PATCH
    - Nombre
    - Categoria
    - Descripcion
    - Composicion
    - Precio de miembro
    - Precio publico
    - Estado
    - Beneficios
4. las imagenes del producto se mostraran en grid y se podra realizar las siguientes acciones
    - Agregar imagen con el action addProductImageAction que es un POST con el key 'image' y el valor es el archivo de imagen
    - Actualizar imagen que abrira un modal con un formulario donde permitira cambiar isMain y  order con el action updateProductImageAction que es un PATCH
    - Eliminar imagen con el action deleteProductImageAction que es un DELETE
6. en la parte del stock se podra agrear stock mediante un formulario que tendra los siguientes campos
    - actionType (select) con las opciones 'INCREASE' y 'DECREASE'
    - quantity
    - description siendo este un campo opcional
   y se enviara al action addStockProductAction que es un POST

si se realiza alguna accion mostrar la alerta en sonner una vez culminada la accion
si se actualiza algo del producto se debe volver a consultar getProductAdmin, si se actualiza el stock se debe volver a consultar getHistoryProductStock
